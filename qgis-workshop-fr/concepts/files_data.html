Fichiers et données

Les données SIG sont principalement des données vectorielles et des données raster

Les données vectorielles

Les données vectorielles sont utilisées pour représenter des objets comme les points, les lignes et les polygones par un ensemble de coordonnées graphiques. Elles incluent généralement des valeurs attributs pour chaque objet – comme par exemple un numéro de route, le nom d’un village ou la population d’une région.

Les données vectorielles sont disponibles sous plusieurs formats, par example :

  • .SHP: Le format de fichier de formes d’ESRI, c’est un format standard pour le stockage de données géographiques et d'échange (c’est un ensemble de fichiers avec le même nom mais des extensions différentes dont une en .shp)
  • .KML/.KMZ: “Keyhole Markup Language”, un langage XML utilisé par Google Earth et Google Maps
  • .TAB : le format de MapInfo, comme shapefiles, un ensemble de fichiers dont un est .TAB
  • un tableau de base de données
  • des fichiers .CSV (comma separated values)

Les données vectorielles peuvent être manipulées, extraites, publiées, personnalisées, traitées et combinées en utilisant les outils SIG ou même parfois, avec des outils de traitement de données standards comme les tableurs ou les bases de données.

Les données raster

Les données rasters sont des images – par exemple des satellites, des photographies aériennes ou des images de cartes. Elles sont souvent utilisées pour le fond des cartes mais peuvent aussi être analysées au niveau des pixels notamment pour identifier l’utilisation des terres. Nous n’allons pas travailler sur les données raster dans cet atelier.

Les fichiers de projet

Les outils spécifiques de SIG ont leur propre format pour les fichiers qui définissent les cartes qu’ils produisent – ces fichiers contiennent des pointeurs vers les données, les styles, etc. Par exemple, le format ESRI utilise des fichiers .mxd, QGIS utilise des fichiers .qgs – intitulés Fichier de projet dans QGIS.

A noter: un fichier de projet QGIS est en fait un fichier XML qui contient des références de couches, qui met en forme et qui configure presque tout en utilisant l’interface QGIS. Cela peut être utile pour corriger des lieux dans un fichier par exemple en utilisant la fonctionnalité rechercher et remplacer.
