Architecture et extensions

QGIS a un programme de base avec une option pour étendre ses fonctionnalités en installant des extensions ou plugins. Certains sont automatiquement installés avec le programme de base, alors que d’autres sont disponibles dans le « QGIS plugins repository ». Il contient des centaines de extensions, dont certains encore en phase expérimentale - gardez donc à l’esprit qu’ils peuvent réagir différemment de ce que vous auriez voulu.

Quelques plugins pratiques :

  • OpenLayers : pour apporter les fonds des cartographies provenant d’OpenStreetMap, de Google, de Bing, etc.
  • InaSAFE : un outil pour modéliser l’impact des risques naturels.
  • HeatMap : créer une carte thermique à partir des données de points.
  • TimeManager : créer des animations à partir des données en séries chronologiques.
  • GroupStats : une sorte de mini créateur de tableaux pivot.
  • Table Manager : un outil très simple d’utilisation pour ajouter, modifier ou supprimer des champs.

Les extension sont gérés par le menu Extensions de QGIS – vous pouvez alors choisir ceux que vous voulez installer et ceux à activer.

L’option Paramètres vous permet de choisir si vous voulez afficher les extensions expérimentaux dans la liste des extensions disponibles et aussi d’afficher les mises à jour disponibles au chargement de QGIS – c’est recommandé car plusieurs extensions sont mis à jour régulièrement (surtout ceux en phase expérimentale).

Exercice : Ouvrir QGIS, configurer l’interface comme décrit plus tôt, et choisir quelques extensions à installer (dont OpenLayers). A noter que la page information de chaque extension doit également vous indiquer comment y accéder dans QGIS – parfois c’est dans le menu des extensions mais parfois les extensions créent leur propre menu et/ou barre d’outil.
